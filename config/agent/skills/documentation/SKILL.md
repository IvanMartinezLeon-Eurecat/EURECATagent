# Skill de Documentación

## Descripción General
Experto en comunicación técnica y documentación. Se enfoca en la claridad, facilidad de búsqueda y utilidad de la información para el equipo de ingeniería.

## Gotchas (Reglas Críticas)
- **Documentación Obsoleta**: Si el código cambia y la doc no, bórrala o márcala como `[STALE]`.
- **Ambigüedad**: Evita "tal vez" o "podría". Usa imperativos claros.
- **Formato**: Usa siempre Markdown estándar con resaltado de sintaxis para bloques de código.

## Flujo de Trabajo (Workflow)
- [ ] **Fase 1: Identificación de Audiencia**
  - Determinar si el documento es para un nuevo integrante (onboarding) o para un experto (referencia API).
- [ ] **Fase 2: Estructura del Documento**
  - Crear un índice (Table of Contents) si el documento es largo.
  - Seguir el patrón: Contexto -> Uso -> Ejemplos -> Referencia Técnica.
- [ ] **Fase 3: Redacción y Visualización**
  - Escribir en lenguaje técnico profesional.
  - Añadir diagramas Mermaid para flujos complejos.
- [ ] **Fase 4: Validación de Enlaces**
  - Verificar que todos los links a archivos locales y URLs externas funcionan.

## Criterios de Éxito
- Cero ambigüedad para nuevos desarrolladores.
- Descripciones completas de APIs y módulos.
- Formato visual impecable y profesional.
